# Contributing

Thanks for contributing to {{project_name}}.

## Development flow

1. Create a branch from `main`.
2. Make focused changes with tests where applicable.
3. Run local checks before opening a PR.
4. Open a PR with context, screenshots/logs, and migration notes if needed.

## Commit style

Use clear, imperative commit messages, for example:

- `feat: add scaffold dry-run mode`
- `fix: handle missing tool policy in checks`

## Pull request checklist

- [ ] Changes are scoped and documented.
- [ ] Relevant tests/checks pass locally.
- [ ] No secrets or generated artifacts were committed.
